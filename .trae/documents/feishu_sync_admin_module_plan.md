# 飞书文档同步后台管理模块实现计划

## 需求分析

在后台管理页面新增飞书文档同步配置管理功能，包含：
1. 配置的增删改查
2. 同步测试功能
3. 同步状态展示

## 项目现状

已有的基础设施：
- ✅ Prisma 模型 `FeishuSyncConfig` 已定义
- ✅ 基础的飞书同步API (`/api/sync/feishu`)
- ✅ 飞书客户端和同步服务
- ✅ 后台管理页面框架

## 实现方案

### 1. 完善飞书同步API

需要在现有 API 基础上添加：
- POST 创建新配置功能（目前只支持 sync/sync-all 操作）
- 保持现有的 GET/PUT/DELETE 功能

### 2. 修改后台管理页面

在 `app/admin/page.tsx` 中：

**UI 变更：**

1. **侧边栏菜单：**
   - 新增"飞书同步"菜单项
   - 支持在"报告管理"和"飞书同步"之间切换

2. **内容区域：**
   - 添加 Tab 切换（报告管理 / 飞书同步）
   - 飞书同步配置列表
   - 配置创建/编辑表单
   - 测试同步按钮
   - 显示最后同步时间和状态

3. **配置表单字段：**
   - 启用/禁用开关
   - App ID (必填)
   - App Secret (必填)
   - 文件夹 Token (可选)
   - 同步时间 (HH:MM 格式，默认 09:00)
   - 报告类型 (day/week/month)
   - 主题分类

### 3. 文件变更清单

| 文件 | 变更类型 | 说明 |
|------|---------|------|
| `app/api/sync/feishu/route.ts` | 修改 | 添加创建配置功能 |
| `app/admin/page.tsx` | 修改 | 添加飞书同步管理UI |

## 实现步骤

### 步骤 1：完善飞书同步 API

修改 `app/api/sync/feishu/route.ts`：
- POST 方法新增 `action === 'create'` 来处理创建配置
- 验证必填字段

### 步骤 2：重构后台管理页面

修改 `app/admin/page.tsx`：
1. 添加状态管理：当前视图模式（报告管理/飞书同步）
2. 添加飞书配置相关状态
3. 修改侧边栏，添加菜单切换
4. 添加飞书同步配置列表 UI
5. 添加配置创建/编辑模态框
6. 添加测试同步功能
7. 显示同步状态和历史

## 技术要点

- 使用现有的 UI 组件和样式风格保持一致
- 表单验证确保必填字段
- 加载状态和错误处理
- 敏感信息（App Secret）显示时适当隐藏或脱敏

## 风险考虑

- API 权限验证已通过 NextAuth session 处理
- 错误信息友好展示
- 配置数据安全性
